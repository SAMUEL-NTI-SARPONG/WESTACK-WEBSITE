GitHub Deployment Guide

Push your West ACK website to GitHub and deploy with GitHub Pages

1. Prerequisites ONE-TIME

You need these installed before starting:
  • Git — Download from git-scm.com and install (keep all defaults during installation)
  • GitHub Account — Create one free at github.com/join

Verify Git is installed

Open Command Prompt or VS Code Terminal (Ctrl + `) and type:

git --version

You should see something like git version 2.43.0. If you get an error, restart your computer after installing Git.

Configure Git with your identity (one-time)

# Replace with YOUR name and email (use the same email as your GitHub account)
git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

2. First-Time Push to GitHub ONE-TIME

1

Create a new repository on GitHub

  1. Go to github.com/new
  2. Repository name: westack-website (or any name you prefer)
  3. Description: West ACK Engineering Ltd. - Official Website
  4. Select Public (required for free GitHub Pages hosting)
  5. Do NOT check "Add a README file" or any other options
  6. Click "Create repository"
2

Open Terminal in VS Code

Press Ctrl + ` to open the terminal. Make sure you're in the website folder:

# Navigate to your website folder
cd "C:\Users\Samuel\OneDrive\Desktop\PROJECT WESTACK\westack-website"
3

Initialize Git and make your first commit

# Initialize a new Git repository
git init

# Add ALL files to staging
git add .

# Create your first commit
git commit -m "Initial commit: West ACK Engineering website"
4

Connect to GitHub and push

# Set the main branch name
git branch -M main

# Connect to your GitHub repo (replace YOUR-USERNAME with your GitHub username)
git remote add origin https://github.com/YOUR-USERNAME/westack-website.git

# Push everything to GitHub
git push -u origin main
First time? A browser window will pop up asking you to sign in to GitHub. Sign in and authorize Git. This only happens once.
5

Verify on GitHub

Go to https://github.com/YOUR-USERNAME/westack-website — you should see all your website files there!

3. Deploy with GitHub Pages ONE-TIME

GitHub Pages gives you free hosting with a URL like your-username.github.io/westack-website

1

Go to repository Settings

On your GitHub repo page, click Settings (top menu bar, far right)

2

Navigate to Pages

In the left sidebar, click Pages (under "Code and automation")

3

Configure the source

  • Source: Select "Deploy from a branch"
  • Branch: Select main
  • Folder: Select / (root)
  • Click Save
4

Wait 2-5 minutes

GitHub will build and deploy your site. Refresh the Settings > Pages page — you'll see a green banner with your live URL:

https://YOUR-USERNAME.github.io/westack-website/
Your website is now live! Share the URL with anyone. Every time you push changes (see section 4), the site updates automatically within a few minutes.

4. Pushing Future Updates EVERY TIME

After making any changes to your website files, use these 3 commands to push them live:

# 1. Navigate to the project folder (if not already there)
cd "C:\Users\Samuel\OneDrive\Desktop\PROJECT WESTACK\westack-website"

# 2. Stage all changed files
git add .

# 3. Commit with a description of what you changed
git commit -m "Updated service images and contact info"

# 4. Push to GitHub (site auto-updates in ~2 mins)
git push
Tip: Write meaningful commit messages so you can track what changed:
git commit -m "Added new portfolio images"
git commit -m "Updated phone number in footer"
git commit -m "Replaced carousel images with real project photos"

Quick Reference — The 3 Commands You'll Always Use

git add .
git commit -m "your message here"
git push

5. Custom Domain (Optional)

To use your own domain (e.g., www.westackengineering.com) instead of the github.io URL:

1

Buy a domain

Purchase from Namecheap, Google Domains, or any registrar.

2

Create a CNAME file

In your westack-website folder, create a file named exactly CNAME (no extension) containing only your domain:

www.westackengineering.com
3

Configure DNS at your domain registrar

Add these DNS records:

Type: CNAME
Host: www
Value: YOUR-USERNAME.github.io

# For apex domain (without www), add these A records:
Type: A → 185.199.108.153
Type: A → 185.199.109.153
Type: A → 185.199.110.153
Type: A → 185.199.111.153
4

Enable in GitHub Settings

Go to Settings > Pages > Custom domain, enter your domain, check "Enforce HTTPS", and Save.

5

Push and wait

git add .
git commit -m "Added custom domain"
git push

DNS changes can take up to 24-48 hours to propagate worldwide.

6. Troubleshooting

Common Issues

Problem Solution
'git' is not recognized Restart VS Code/PC after installing Git. If still failing, reinstall Git and check "Add to PATH" during setup.
Push rejected / authentication error Make sure you signed into GitHub when the browser popup appeared. Try: git push again.
GitHub Pages shows 404 Make sure index.html is at the ROOT of your repo (not inside a subfolder). Wait 5 minutes and try again.
CSS / images not loading on live site Check that file paths in your HTML are relative (e.g., css/style.css not /css/style.css or C:\...). Clear browser cache (Ctrl+Shift+R).
Changes not showing after push GitHub Pages can take 2-10 minutes to update. Hard refresh your browser: Ctrl+Shift+R
fatal: not a git repository You're in the wrong folder. Run cd to navigate to westack-website first.

Useful Git commands

# Check status of changed files
git status

# See history of commits
git log --oneline

# Undo all uncommitted changes (careful!)
git checkout .

# See what changed in files
git diff
