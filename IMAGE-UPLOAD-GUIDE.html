3. Step-by-Step: How to Replace Any Image

Method A — Simple File Replacement (recommended)

1

Prepare your new image

Resize and crop it to the recommended dimensions shown in the Master Table above.

2

Rename it to match the existing filename exactly

For example, if you want to replace the first carousel image, your file must be named carousel-1.JPG (case-sensitive on some hosting).

3

Copy it into the img/ folder, replacing the old file

Navigate to westack-website\img\ in File Explorer and paste. Click "Replace" when prompted.

4

Refresh your browser

Press Ctrl + Shift + R (hard refresh) to see the new image. Browsers cache old images, so a normal refresh may show the old one.

Important: File names are case-sensitive on most web hosts. carousel-1.JPG is different from carousel-1.jpg. Match the exact capitalization shown in the Master Table.

Method B — Using a Differently Named File

If your image has a different name and you want to keep it:

  1. Place the image in westack-website/img/
  2. Open the HTML file where the image is used (see Master Table)
  3. Find the <img src="img/old-name.jpg"> tag
  4. Change old-name.jpg to your new filename
  5. Save the HTML file and refresh the browser

5. Image Specifications and Best Practices

Dimensions Summary

Logo: 200 x 60 px, PNG transparent

Carousel/Hero: 1920 x 1080 px, JPG

Service cards: 600 x 400 px, JPG

Portfolio: 600 x 400 px, JPG

About photo: 800 x 600 px, JPG

Service images: 600 x 400 px, JPG

Civil gallery: 800 x 600 px, JPG

Service banner: 1200 x 600 px, JPG

File Size Guidelines

Carousel images: 200-500 KB each

Service/Portfolio: 50-200 KB each

Logos: 10-50 KB each

Total page weight target: under 3 MB per page

Compress images at TinyPNG or Squoosh before uploading.

Best Practices

  • Consistency: All service card images should have the same dimensions and similar colour tone.
  • Professional photos: Use real project photos. Avoid stock images of unrelated sites.
  • Landscape orientation: All images except the logo and client logos should be landscape (wider than tall).
  • No text on photos: The website adds text overlays automatically — keep photos clean.
  • PNG for logos, JPG for photos: PNG preserves transparency; JPG is smaller for photographs.
  • Alt text: The HTML already includes descriptive alt text for accessibility.
